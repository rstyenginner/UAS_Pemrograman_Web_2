# LOGBOOK MINGGU 1

# Sistem Manajemen Buku Digital Berbasis REST API

## 1. Pencapaian Minggu Ini

Pada minggu pertama, tim telah menentukan tema proyek yaitu "Sistem Manajemen Buku Digital Berbasis REST API". Sistem ini dirancang untuk mengelola dua jenis buku, yaitu e-book PDF dan buku fisik. E-book dapat dibaca langsung melalui sistem, sedangkan buku fisik dapat diajukan peminjaman oleh member.

Selain itu, telah dibuat rancangan awal berupa deskripsi sistem, tujuan sistem, aktor sistem, kebutuhan fungsional, kebutuhan non-fungsional, alur sistem, rancangan database awal, rancangan API awal, dan pembagian peran tim.

## 2. Bukti Kerja

Bukti kerja minggu ini:

1. Dokumen SRS awal.
2. Draft flowchart alur member membaca e-book.
3. Draft flowchart alur member meminjam buku fisik.
4. Draft flowchart admin mengelola buku.
5. Draft use case member dan admin.
6. Draft pembagian tugas tim.
7. Rancangan awal database.
8. Rancangan awal endpoint REST API.

## 3. Kendala

Kendala yang ditemukan:

1. Masih perlu memastikan framework backend yang akan digunakan, apakah mengikuti contoh tugas menggunakan CodeIgniter 4 atau menggunakan Laravel apabila diizinkan.

## 4. Solusi Sementara

Solusi sementara yang dilakukan:

1. Sistem dirancang terlebih dahulu secara umum berbasis REST API agar dapat diterapkan pada CodeIgniter 4 maupun Laravel.
2. Role aplikasi dibuat sederhana, yaitu Admin dan Member.
3. Fitur utama difokuskan pada katalog buku, baca PDF, peminjaman buku fisik, kelola buku, dan kelola peminjaman.
4. Dokumentasi awal dibuat dalam format Markdown agar mudah dibaca di GitHub.
5. Diagram awal dibuat menggunakan visio di dalam file Markdown.

## 5. Target Minggu Depan

Target minggu depan:

1. Membuat ERD final.
2. Membuat struktur database final.
3. Membuat migration dan seeder.
4. Setup repository GitHub.
5. Setup folder monorepo.
6. Membuat desain UI awal.
7. Setup project backend.
8. Setup project frontend.
9. Membuat endpoint API awal untuk auth, kategori, dan buku.

## 6. Catatan Tambahan

Fokus pengerjaan minggu pertama adalah tahap analisis sistem. Tahap ini menjadi dasar sebelum masuk ke pembuatan database, backend, frontend, dan UI/UX. Dengan adanya dokumen SRS, flowchart, use case, dan pembagian tugas.
